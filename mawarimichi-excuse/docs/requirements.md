# 要件定義書

## 1. 目的
遅刻や未提出などの日常的な失敗を、AIで公文書風の壮大な言い訳に変換するジョークWebアプリを提供する。

## 2. 対象ユーザー
- 学生
- ハッカソン参加者
- ジョークコンテンツをSNS投稿したいユーザー

## 3. 機能要件
- 申請フォーム入力
  - 申請者氏名
  - 事象（遅刻/未提出）
  - 世俗的理由
- AI言い訳生成
  - API経由でJSONを返却
  - ローディング演出表示
- 証明書表示
  - 公文書風レイアウト
  - 認定ランク表示
  - 角印表示
- 画像ダウンロード
  - プレビュー領域をPNG保存

## 4. 非機能要件
- ブラウザ上で3秒以内にフォーム描画
- API失敗時にユーザー向けエラー文言を返す
- 秘密情報をリポジトリへ含めない
- モバイル幅でもUI崩れを最小化

## 5. 制約
- ジョーク用途であり、公的効力なし
- LLM APIの可用性・レート制限に依存
